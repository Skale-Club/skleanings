<objective>
## Goal
Add a nullable `userId` FK on `bookings` to link bookings to authenticated client accounts. When an authenticated client submits a booking, the server attaches their `userId` automatically. Guest bookings remain fully supported with `userId = null`.

## Purpose
This is the data model that makes the client portal possible. Without `bookings.userId`, the portal has no way to scope booking history to a specific account. Phase 11 (self-service API) and phase 12 (portal UI) both depend on this ownership link existing.

## Output
- `bookings.userId` nullable FK column in schema (requires `npm run db:push`)
- `getBookingsByUserId(userId)` storage method (used by phase 11 client API)
- POST `/api/bookings` attaches `userId` from auth token when caller is an authenticated client; guest callers unaffected
</objective>
