<objective>
## Goal
Add `client` as a first-class role across the server middleware, frontend AuthContext, login redirect logic, and App.tsx route guard â€” so a client user can sign in and land at `/account` with all other roles blocked from that area.

## Purpose
v1.0 introduces a self-service customer portal. Before any portal UI or API can exist, the `client` role must be wired in as a recognized value at every role-boundary: auth middleware, TypeScript types, login redirect decision, and App-level route protection. This is the foundation all later portal plans depend on.

## Output
- `requireClient` middleware in `server/lib/auth.ts`
- `UserRole` type widened to include `'client'` + `isClient` value in `AuthContext`
- Login page redirects `client` role to `/account` instead of falling through to `/`
- `/account/*` route segment in `App.tsx` â€” guarded to client role, placeholder page component until phase 12
</objective>
