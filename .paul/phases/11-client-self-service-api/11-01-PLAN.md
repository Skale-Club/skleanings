<objective>
## Goal
Add `GET /api/client/me`, `PATCH /api/client/me`, `GET /api/client/bookings`, and `GET /api/client/bookings/:id` — all protected by `requireClient` — so authenticated clients can read their profile and view their own booking history.

## Purpose
Phase 1 of the client portal API. Gives the frontend everything it needs to render the account shell and booking list before any mutation endpoints exist. Also implements the email-match legacy lookup so pre-ownership bookings (userId = null, same email) appear in the client's history.

## Output
- New `server/routes/client.ts` router
- `getClientBookings(userId, email)` storage method (merges userId + email-match rows, deduplicated)
- Router mounted at `/api/client` in `server/routes.ts`
</objective>

    **1. Add `getClientBookings` to `server/storage.ts`:**

    Add to `IStorage` interface:
    ```ts

    ```

    Add to `DatabaseStorage`:
    ```ts
    async getClientBookings(userId: string, email: string): Promise<Booking[]> {
      // Two queries: userId-linked + email-match legacy (userId IS NULL + email matches)
      const [byUserId, byEmail] = await Promise.all([
        db.select().from(bookings).where(eq(bookings.userId, userId)),
        db.select().from(bookings).where(
          and(isNull(bookings.userId), eq(bookings.customerEmail, email))
        ),
      ]);
      // Merge, deduplicate by id, sort by createdAt desc
      const seen = new Set<number>();
      const merged = [...byUserId, ...byEmail].filter(b => {
        if (seen.has(b.id)) return false;
        seen.add(b.id);
        return true;
      });
      merged.sort((a, b) => new Date(b.createdAt!).getTime() - new Date(a.createdAt!).getTime());
      return merged;
    }
    ```

    Import `isNull`, `and` from drizzle-orm if not already imported at top of storage.ts.

    **2. Add booking endpoints to `server/routes/client.ts`:**

    **GET /api/client/bookings:**
    - Apply `requireClient`
    - Call `getAuthenticatedUser(req)` to get user
    - Call `storage.getClientBookings(user.id, user.email ?? '')`
    - Return 200 with array

    **GET /api/client/bookings/:id:**
    - Apply `requireClient`
    - Call `getAuthenticatedUser(req)` to get user
    - Call `storage.getBooking(Number(req.params.id))`
    - If not found: 404
    - Ownership check: `booking.userId === user.id || (booking.userId === null && booking.customerEmail === user.email)`
    - If not owner: 403 `{ message: "Forbidden" }`
    - Return 200 with booking

    **3. Mount router in `server/routes.ts`:**
    - Add `import clientRouter from "./routes/client";` with other router imports
    - Add `app.use("/api/client", clientRouter);` after the staff router line

    Avoid: exposing admin-only booking fields differently — return the booking as-is (same shape as admin sees; the frontend will decide what to render).
