## Purpose

<acceptance_criteria>

## AC-1: Cancel — ownership + status + date-window
```gherkin
Given a client who owns booking 42 (status=confirmed, bookingDate tomorrow)
When POST /api/client/bookings/42/cancel
Then 200 { success: true } and booking.status = 'cancelled'

Given booking 42 with status='cancelled'
When POST /api/client/bookings/42/cancel
Then 400 { message: "Booking cannot be cancelled" }

Given booking 42 with bookingDate = today or in the past
When POST /api/client/bookings/42/cancel
Then 400 { message: "Cannot cancel past or same-day bookings" }

Given booking 42 owned by a different client
When POST /api/client/bookings/42/cancel
Then 403 Forbidden
```

## AC-2: Reschedule — ownership + availability + update
```gherkin
Given a client who owns booking 42 (status=confirmed, bookingDate tomorrow)
When POST /api/client/bookings/42/reschedule with { bookingDate, startTime, endTime } for a future free slot
Then 200 with updated booking

Given the requested new slot is already taken
When POST /api/client/bookings/42/reschedule
Then 409 { message: "Requested time slot is not available" }

Given missing bookingDate, startTime, or endTime
When POST /api/client/bookings/42/reschedule
Then 400 validation error
```

<task type="auto">
  <name>Task 1: Cancel endpoint</name>
  <files>server/routes/client.ts</files>
  <action>
    Append to `server/routes/client.ts` (before `export default router`):

    Import `checkAvailability` from `../lib/availability` at the top of the file.

    **POST /api/client/bookings/:id/cancel:**
    - Apply `requireClient`
    - Get user from `(req as any).user`
    - Load booking via `storage.getBooking(Number(req.params.id))`; 404 if not found
    - Ownership check: `booking.userId === user.id || (booking.userId === null && booking.customerEmail === user.email)`; 403 if false
    - Status check: if booking.status !== 'pending' && booking.status !== 'confirmed' → 400 `{ message: "Booking cannot be cancelled" }`
    - Date-window check: compare booking.bookingDate string (YYYY-MM-DD) > today string. Get today via `new Date().toISOString().slice(0, 10)`. If booking.bookingDate <= today → 400 `{ message: "Cannot cancel past or same-day bookings" }`
    - Call `storage.updateBookingStatus(booking.id, 'cancelled')`
    - Return 200 `{ success: true }`

    Avoid: importing or calling any GHL sync here — that is plan 11-03.
  </action>
  <verify>
    File compiles (no obvious syntax errors); import of checkAvailability present at top.
  </verify>
  <done>AC-1 satisfied: ownership, status, date-window guards all present; status set to cancelled on success</done>

<task type="auto">
  <name>Task 2: Reschedule endpoint</name>
  <files>server/routes/client.ts</files>
  <action>
    Append to `server/routes/client.ts` (before `export default router`):

    **POST /api/client/bookings/:id/reschedule:**
    - Apply `requireClient`
    - Validate body with Zod: `z.object({ bookingDate: z.string(), startTime: z.string(), endTime: z.string() })` — return 400 on ZodError
    - Get user from `(req as any).user`
    - Load booking via `storage.getBooking(Number(req.params.id))`; 404 if not found
    - Ownership check: same pattern as cancel; 403 if fails
    - Status check: if booking.status !== 'pending' && booking.status !== 'confirmed' → 400 `{ message: "Booking cannot be rescheduled" }`
    - Date-window check: new bookingDate must be > today (`validatedData.bookingDate > today`); if not → 400 `{ message: "New booking date must be in the future" }`
    - Availability check: `await checkAvailability(validatedData.bookingDate, validatedData.startTime, validatedData.endTime, booking.id)` — pass `booking.id` as excludeBookingId so the booking doesn't conflict with itself. If not available → 409 `{ message: "Requested time slot is not available" }`
    - Call `storage.updateBooking(booking.id, { bookingDate: validatedData.bookingDate, startTime: validatedData.startTime, endTime: validatedData.endTime })`
    - Return 200 with updated booking

    Avoid: notifying GHL here — that is plan 11-03.
  </action>

  </verify>
