<objective>
## Goal
Implement backend reconnect detection: when a staff member's Google Calendar token fails to refresh, mark it as `needsReconnect = true`, record `lastDisconnectedAt`, and send an SMS alert via Twilio to the configured admin numbers.

## Purpose
Staff Google Calendar tokens expire or get revoked without warning. Currently the system silently returns no busy times, so double-bookings can occur. This plan closes that gap: the system detects the failure, marks the record, notifies the admin via SMS, and exposes the status for the frontend banner (Plan 05-02).

## Output
- `server/storage.ts` — 2 new methods: `markCalendarNeedsReconnect`, `getAllCalendarStatuses`
- `server/lib/google-calendar.ts` — `getValidAccessToken` triggers mark + SMS on refresh failure
- `server/routes/staff.ts` — 2 new endpoints: `GET /api/staff/calendar/all-statuses`, `POST /api/staff/:id/calendar/clear-reconnect`
</objective>
