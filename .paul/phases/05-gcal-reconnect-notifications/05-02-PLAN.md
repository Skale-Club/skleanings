<objective>
## Goal
Build the "Take Action" in-app banner that appears in the admin dashboard when any staff member's Google Calendar is disconnected, plus update the CalendarTab to show reconnect state and clear the flag after successful re-auth.

## Purpose
Mirrors the GoHighLevel reconnect alert UX: a persistent banner at the top of the admin area warns when integrations need attention, with a modal listing affected staff and direct reconnect links.

## Output
- `client/src/components/admin/CalendarReconnectBanner.tsx` — new component: banner + modal
- `client/src/pages/Admin.tsx` — banner wired into admin layout
- `client/src/components/admin/StaffManageDialog.tsx` — CalendarTab shows reconnect warning + clears flag on re-auth
- `server/routes/staff.ts` — OAuth callback clears needsReconnect automatically
</objective>

<task type="auto">
  <name>Task 2: Update CalendarTab for reconnect state + auto-clear on OAuth callback</name>
  <files>client/src/components/admin/StaffManageDialog.tsx, server/routes/staff.ts</files>
  <action>
    **Update CalendarTab in StaffManageDialog.tsx:**

    1. Modify the existing calendar status query to also fetch `needsReconnect` from the status endpoint.
       - Current endpoint `GET /api/staff/:id/calendar/status` needs to also return `needsReconnect`.
       - Update the server route: in the existing `/:id/calendar/status` handler (line 188), when record exists, also include `needsReconnect: record.needsReconnect` in the response JSON.

    2. Add a third UI state to CalendarTab (currently has "connected" and "disconnected"):
       - **Needs Reconnect state** (when `status?.connected === false && status?.needsReconnect === true`... OR more precisely: when the record exists but needsReconnect is true):
         - Show an amber/yellow Alert with AlertTriangle icon.
         - Text: "Calendar disconnected — reconnect to resume availability sync"
         - "Reconnect" button (same `<a>` link as the existing connect button, pointing to `/api/staff/{staffId}/calendar/connect`).
         - Also show a "Disconnect completely" link/button below (existing delete flow).

    3. State priority in CalendarTab render:
       - `needsReconnect === true` → Show reconnect warning state
       - `connected === true` → Show connected state (existing)
       - Otherwise → Show disconnected/connect state (existing)

    **Update OAuth callback to auto-clear needsReconnect:**

    In `server/routes/staff.ts`, the OAuth callback handler at `GET /api/staff/calendar/callback` (around line 92):
    - After `exchangeCodeForTokens(code, staffId)` succeeds, add: `await storage.clearCalendarNeedsReconnect(staffId)`.
    - This ensures that after successful re-authentication, the needsReconnect flag is cleared automatically — the staff member doesn't need to manually clear it.
    - The `upsertStaffGoogleCalendar` already replaces tokens, but it doesn't touch needsReconnect because that field is omitted from the insert schema.

    **Update per-staff calendar/status endpoint:**

    In `server/routes/staff.ts`, the `GET /:id/calendar/status` handler (line 188):
    - Current response: `{ connected: true, calendarId, connectedAt }`
    - New response: `{ connected: true, calendarId, connectedAt, needsReconnect: record.needsReconnect }`
    - When record doesn't exist: `{ connected: false, needsReconnect: false }`

    Avoid: Breaking existing CalendarTab behavior for connected/disconnected states.
  </action>
  <verify>
    1. `npm run check` passes.
    2. After OAuth callback completes, verify needsReconnect is false in DB.
    3. CalendarTab shows reconnect warning when needsReconnect=true, normal connected state when false.
  </verify>
  <done>AC-3 satisfied: CalendarTab shows reconnect warning. AC-4 satisfied: OAuth callback auto-clears needsReconnect.</done>
