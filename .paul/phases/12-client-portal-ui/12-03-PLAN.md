<acceptance_criteria>

## AC-1: Cancel dialog confirms before cancelling
```gherkin
Given a client clicks Cancel on an eligible booking
When the cancel dialog opens
Then it shows the booking date and asks for confirmation
And clicking Confirm calls POST /api/client/bookings/:id/cancel
```

## AC-2: Cancel success updates the list
```gherkin
Given a client confirms cancellation in the dialog
When POST /api/client/bookings/:id/cancel succeeds
Then the dialog closes, a success toast shows, and the booking list refreshes with the updated status
```

## AC-3: Reschedule dialog shows date picker and available slots
```gherkin
Given a client clicks Reschedule on an eligible booking
When the reschedule dialog opens
Then a date input (min=today) is shown
And selecting a date fetches and displays available time slots via GET /api/availability
```

## AC-4: Reschedule submission sends correct payload
```gherkin
Given a client selects a new date and time slot in the reschedule dialog
When they click Confirm Reschedule
Then POST /api/client/bookings/:id/reschedule is called with { bookingDate, startTime, endTime }
Where endTime is computed from startTime + booking.totalDurationMinutes
```

## AC-5: Reschedule success updates the list
```gherkin
Given a client confirms a reschedule
When POST /api/client/bookings/:id/reschedule succeeds
Then the dialog closes, a success toast shows, and the booking list refreshes with the new date/time
```

## AC-6: API errors shown as toasts
```gherkin
Given the cancel or reschedule API returns an error (e.g. booking past window, already cancelled)
When the error is received
Then a descriptive error toast is shown and the dialog stays open for retry or dismissal
```

  <name>Task 2: Build RescheduleBookingDialog and wire Reschedule button</name>
  <files>client/src/components/account/RescheduleBookingDialog.tsx, client/src/components/account/BookingsSection.tsx</files>
  <action>
    Create `client/src/components/account/RescheduleBookingDialog.tsx`:
    
    Props: `{ booking: Booking; open: boolean; onClose: () => void; onSuccess: () => void }`
    
    Use shadcn Dialog (import from '@/components/ui/dialog'):
    - Dialog, DialogContent, DialogHeader, DialogTitle, DialogFooter
    
    Internal state: `selectedDate: string` (YYYY-MM-DD, init ""), `selectedTime: string` (HH:MM, init "")
    
    Date input:
    - HTML `<input type="date" min={today} value={selectedDate} onChange={...} />`
    - today = new Date().toISOString().slice(0, 10)
    - Reset selectedTime to "" whenever selectedDate changes
    - Style: border rounded px-3 py-2 w-full
    
    Availability fetch (useAvailability hook from '@/hooks/use-booking'):
    - `useAvailability(selectedDate || undefined, booking.totalDurationMinutes)`
    - No staffId/serviceIds — use booking-level duration only (client doesn't know service breakdown)
    - Shows loading spinner while fetching
    - Renders slot buttons: filter slots where slot.available, render each as a pill button
      - Selected: bg-primary text-white; unselected: border hover:bg-slate-50
    - If no slots for date: "No available slots for this date."
    
    endTime computation helper:
    ```ts
    function addMinutes(time: string, minutes: number): string {
      const [h, m] = time.split(':').map(Number);
      const total = h * 60 + m + minutes;
      return `${String(Math.floor(total / 60) % 24).padStart(2, '0')}:${String(total % 60).padStart(2, '0')}`;
    }
    ```
    
    Mutation (useMutation):
    - mutationFn: getAccessToken() →
      fetch POST /api/client/bookings/${booking.id}/reschedule with Authorization header + JSON body:
      { bookingDate: selectedDate, startTime: selectedTime, endTime: addMinutes(selectedTime, booking.totalDurationMinutes) }
    - onSuccess: toast "Booking rescheduled", onSuccess()
    - onError: toast error message, do NOT call onClose
    
    Submit button:
    - Disabled when !selectedDate || !selectedTime || isPending
    - Text: isPending ? "Rescheduling..." : "Confirm Reschedule"
    
    Reset state when dialog closes (useEffect on open: when open → false, reset selectedDate/selectedTime)
    
    In `BookingsSection.tsx`:
    - Add state: `const [rescheduleBooking, setRescheduleBooking] = useState<Booking | null>(null)`
    - Wire Reschedule button onClick: `() => setRescheduleBooking(booking)`
    - Render alongside CancelBookingDialog:
      ```tsx
      {rescheduleBooking && (
        <RescheduleBookingDialog
          booking={rescheduleBooking}
          open={!!rescheduleBooking}
          onClose={() => setRescheduleBooking(null)}
          onSuccess={() => {
            setRescheduleBooking(null);
            queryClient.invalidateQueries({ queryKey: ['/api/client/bookings'] });
          }}
        />
      )}
      ```
    - Import RescheduleBookingDialog
    
    Avoid: passing staffId/serviceIds to useAvailability (booking record has staffMemberId but client API doesn't guarantee it; duration-only is safe and consistent); calling onClose on API error
  </action>
