<acceptance_criteria>

## AC-1: Client login page at /account/login
```gherkin
Given an unauthenticated user visits /account/login
When the page loads
Then a login form renders with company branding, email field, password field, and sign-in button
```

## AC-2: Client login redirects to account on success
```gherkin
Given a client-role user submits valid credentials at /account/login
When authentication completes
Then they are redirected to /account (not /admin or /staff/settings)
```

## AC-3: Unauthenticated access to /account redirects to /account/login
```gherkin
Given an unauthenticated user visits /account or /account/bookings
When the shell loads
Then they are redirected to /account/login (not /admin/login)
```

## AC-4: AccountShell renders tab navigation for authenticated client
```gherkin
Given an authenticated client visits /account
When the shell loads
Then a header (company name + logout), and two tabs (Profile, My Bookings) are visible
And the active tab reflects the current URL path
```

## AC-5: Non-client roles redirected out of account area
```gherkin
Given an authenticated user with role=admin, user, or staff visits /account
When the shell resolves their role
Then they are redirected to /admin
```
