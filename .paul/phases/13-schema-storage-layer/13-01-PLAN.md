<objective>
## Goal
Add a `notificationLogs` table to `shared/schema.ts` and typed storage methods to `server/storage.ts` so notification sends can be persisted in Phase 14.

## Purpose
All notification sends (Twilio SMS, Telegram, GoHighLevel sync) are currently fire-and-forget with no audit trail. This plan lays the data foundation — table + storage methods — that Phase 14 will use to instrument each send. The log is additive: no existing tables or flows are modified.

## Output
- New `notificationLogs` table in `shared/schema.ts` with insert schema and TypeScript types
- `IStorage` extended with 4 new methods: `createNotificationLog`, `getNotificationLogsByConversation`, `getNotificationLogsByBooking`, `getNotificationLogs`
- `DatabaseStorage` implements all 4 methods
- `npm run db:push` migrates the table cleanly
</objective>

    createNotificationLog(entry: InsertNotificationLog): Promise<NotificationLog>;
    getNotificationLogsByConversation(conversationId: string): Promise<NotificationLog[]>;
    getNotificationLogsByBooking(bookingId: number): Promise<NotificationLog[]>;

    3. Add 4 implementations to `DatabaseStorage` (before the closing `}` of the class):
    ```ts
    async createNotificationLog(entry: InsertNotificationLog): Promise<NotificationLog> {
      const [row] = await db.insert(notificationLogs).values(entry).returning();
      return row;
    }

    async getNotificationLogsByConversation(conversationId: string): Promise<NotificationLog[]> {
      return db
        .select()
        .from(notificationLogs)
        .where(eq(notificationLogs.conversationId, conversationId))
        .orderBy(desc(notificationLogs.sentAt));
    }

    async getNotificationLogsByBooking(bookingId: number): Promise<NotificationLog[]> {
      return db
        .select()
        .from(notificationLogs)
        .where(eq(notificationLogs.bookingId, bookingId))
        .orderBy(desc(notificationLogs.sentAt));
    }

    async getNotificationLogs(filters: {
      channel?: string;
      status?: string;
      trigger?: string;
      search?: string;
      from?: Date;
      to?: Date;
      limit?: number;
      offset?: number;
    }): Promise<NotificationLog[]> {
      const conditions = [];
      if (filters.channel) conditions.push(eq(notificationLogs.channel, filters.channel));
      if (filters.status) conditions.push(eq(notificationLogs.status, filters.status));
      if (filters.trigger) conditions.push(eq(notificationLogs.trigger, filters.trigger));
      if (filters.search) conditions.push(like(notificationLogs.recipient, `%${filters.search}%`));
      if (filters.from) conditions.push(gte(notificationLogs.sentAt, filters.from));
      if (filters.to) conditions.push(lte(notificationLogs.sentAt, filters.to));

      return db
        .select()
        .from(notificationLogs)
        .where(conditions.length ? and(...conditions) : undefined)
        .orderBy(desc(notificationLogs.sentAt))
        .limit(filters.limit ?? 50)
        .offset(filters.offset ?? 0);
    }
    ```

    Import `desc`, `like`, `gte`, `lte`, `and` from "drizzle-orm" — check existing imports at the top of storage.ts and add only what's missing.
    Do NOT modify any existing methods or the IStorage interface beyond adding the new section.
