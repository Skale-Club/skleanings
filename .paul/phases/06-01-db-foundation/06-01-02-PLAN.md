<objective>
## Goal
Add `role` field to the `users` table and `userId` FK to `staffMembers` table; expose `/api/me` endpoint returning the current user's role and linked staffMemberId; extend `requireAdmin` middleware to remain backward compatible while setting `req.user.role`.

## Purpose
The current auth system uses `ADMIN_EMAIL` env var for a binary admin/not-admin gate. Adding `role: 'admin' | 'staff' | 'viewer'` to users enables Phase 4's staff-scoped calendar view, where a logged-in staff user sees only their assignments. `staffMembers.userId` links the staff entity to a Supabase user account, enabling future staff login without disrupting today's admin flow.

## Output
- `users.role` field in `shared/schema.ts` (text, default `'viewer'`, values: `'admin' | 'staff' | 'viewer'`)
- `staffMembers.userId` nullable FK in `shared/schema.ts` (→ users.id)
- `IStorage` + `DatabaseStorage`: `getUserByEmail`, `updateUserRole`, `linkStaffToUser`
- `server/lib/auth.ts`: `requireAdmin` unchanged functionally; adds `req.user.dbRole` from DB lookup
- `GET /api/me` route: returns `{ id, email, role, staffMemberId }` for the authenticated user
- Supabase migration adding the two columns
</objective>

<task type="auto">
  <name>Task 3: Add GET /api/me route</name>
  <files>server/routes.ts</files>
  <action>
    Find the section in `server/routes.ts` where user-related routes live (search for `/api/users` or auth routes). Add the following route in the Users section, protected by `requireAdmin` (since only logged-in admin/staff users call this):

    ```ts
    // GET /api/me — returns current user's role and linked staff member id
    app.get("/api/me", requireAdmin, async (req: Request, res: Response) => {
      try {
        const supabaseUser = (req as any).user;
        if (!supabaseUser?.email) {
          return res.status(401).json({ message: "Not authenticated" });
        }

        // Look up user record in our DB by email
        const dbUser = await storage.getUserByEmail(supabaseUser.email);
        if (!dbUser) {
          // User authenticated with Supabase but not in our users table yet
          // Return minimal profile with viewer role
          return res.json({
            id: supabaseUser.id,
            email: supabaseUser.email,
            role: "viewer",
            staffMemberId: null,
          });
        }

        // Find linked staff member (if any)
        let staffMemberId: number | null = null;
        if (dbUser.id) {
          const allStaff = await storage.getStaffMembers();
          const linked = allStaff.find(s => s.userId === dbUser.id);
          staffMemberId = linked?.id ?? null;
        }

        return res.json({
          id: dbUser.id,
          email: dbUser.email,
          role: dbUser.role ?? "viewer",
          staffMemberId,
        });
      } catch (error) {
        console.error("Error fetching /api/me:", error);
        return res.status(500).json({ message: "Failed to fetch user profile" });
      }
    });
    ```

    Note: `requireAdmin` currently checks ADMIN_EMAIL — this means `/api/me` is only callable by the admin user for now. Phase 4 will add `requireStaff` middleware to broaden access. For now this is intentional scope.
  </action>
