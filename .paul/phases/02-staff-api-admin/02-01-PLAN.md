<task type="auto">
  <name>Task 1: Create server/routes/staff.ts with all 11 endpoints</name>
  <files>server/routes/staff.ts</files>
  <action>
    Create a new file `server/routes/staff.ts` following the exact pattern of `server/routes/faqs.ts`.

    Endpoints to implement:

    **Public endpoints (no auth required — used by booking flow):**
    - `GET /` — list active staff: `storage.getStaffMembers(false)`, ordered by `order`
    - `GET /count` — `{ count: await storage.getStaffCount() }` — MUST be defined before `/:id` to avoid route conflict
    - `GET /:id` — `storage.getStaffMember(id)`, 404 if not found

    **Admin endpoints (requireAdmin middleware):**
    - `POST /` — create: validate with `insertStaffMemberSchema.parse(req.body)`, call `storage.createStaffMember(data)`, return 201
    - `PUT /reorder` — body: `{ updates: { id: number; order: number }[] }`, call `storage.reorderStaffMembers(updates)`, return `{ success: true }` — MUST be defined before `/:id`
    - `PUT /:id` — update: validate with `insertStaffMemberSchema.partial().parse(req.body)`, call `storage.updateStaffMember(id, data)`
    - `DELETE /:id` — call `storage.deleteStaffMember(id)`, return `{ success: true }`

    **Service abilities (admin):**
    - `GET /:id/services` — `storage.getServicesByStaffMember(id)`
    - `PUT /:id/services` — body: `{ serviceIds: z.array(z.number()).parse(req.body.serviceIds) }`, call `storage.setStaffServiceAbilities(id, serviceIds)`, return `{ success: true }`

    **Availability (admin for write, public for read):**
    - `GET /:id/availability` — `storage.getStaffAvailability(id)`
    - `PUT /:id/availability` — body: array of `{ dayOfWeek, startTime, endTime, isAvailable }`, validate each item, call `storage.setStaffAvailability(id, data)`, return saved records

    **Availability item Zod schema (inline in file):**
    ```typescript
    const availabilityItemSchema = z.object({
      dayOfWeek: z.number().int().min(0).max(6),
      startTime: z.string().regex(/^\d{2}:\d{2}$/),
      endTime: z.string().regex(/^\d{2}:\d{2}$/),
      isAvailable: z.boolean(),
    });
    ```

    Error handling: same as faqs.ts — ZodError → 400, catch → 400/500.
    Avoid: Using POST for reorder — use PUT /reorder to match existing patterns.
    Avoid: Defining `/:id` before `/count` or `/reorder` — Express matches routes in order.
  </action>
