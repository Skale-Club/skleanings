<objective>
## Goal
Fix the Google Calendar OAuth flow so staff can connect their calendar with a single button click from /staff/settings, with the callback returning them to the correct page — and so all calendar API calls use authenticated requests.

## Purpose
White-label behavior: admin configures OAuth credentials once (Client ID + Secret + Redirect URI in Integrations). Staff never sees the integrations page — they just click "Connect Google Calendar" from their personal settings, log into Google, and land back on /staff/settings with the calendar connected. Currently the callback always redirects to /admin/staff (a page staff can't access), and CalendarTab uses unauthenticated fetch calls.

## Output
- `server/lib/google-calendar.ts` — getAuthUrl accepts redirectTo param, encoded in OAuth state
- `server/routes/staff.ts` — connect route encodes user role in state; callback routes to correct page
- `client/src/components/admin/CalendarTab.tsx` — uses authenticated requests; connect flow works for both admin and staff contexts
</objective>

    const url = await getAuthUrl(Number(req.params.id), redirectTo);
    res.redirect(url);

<task type="auto">
  <name>Task 2: Frontend — CalendarTab authenticated requests</name>
  <files>client/src/components/admin/CalendarTab.tsx</files>
  <action>
    **Problem:** CalendarTab currently uses plain `fetch()` for the status query and `apiRequest` (no auth header) for disconnect. Staff users authenticate via Supabase JWT (Bearer token), so unauthenticated fetches will return 401.

    **Fix: use `useAdminAuth()` to get the token for all requests.**

    Import `useAdminAuth` and `authenticatedRequest`:
    ```typescript
    import { useAdminAuth } from '@/context/AuthContext';
    import { authenticatedRequest } from '@/lib/queryClient';
    ```

    Inside the component, call:
    ```typescript
    const { getAccessToken } = useAdminAuth();
    ```

    **Status query** — change from plain fetch to authenticated:
    ```typescript
    queryFn: async () => {
      const token = await getAccessToken();
      if (!token) return { connected: false };
      const res = await authenticatedRequest('GET', `/api/staff/${staffId}/calendar/status`, token);
      return res.json();
    },
    ```

    **Disconnect mutation** — change from `apiRequest` to `authenticatedRequest`:
    ```typescript
    mutationFn: async () => {
      const token = await getAccessToken();
      if (!token) throw new Error('Not authenticated');
      return authenticatedRequest('DELETE', `/api/staff/${staffId}/calendar`, token);
    },
    ```

    **Connect button** — the `<a href>` link for connect/reconnect navigates the browser directly to the backend endpoint. This is a full-page redirect, so it won't carry the Bearer token header. Fix: convert to a button with a click handler that passes the token as a query param to the connect endpoint.

    Change the connect `<a>` links to buttons:
    ```typescript
    const handleConnect = async () => {
      const token = await getAccessToken();
      const url = token
        ? `/api/staff/${staffId}/calendar/connect?token=${encodeURIComponent(token)}`
        : `/api/staff/${staffId}/calendar/connect`;
      window.location.href = url;
    };
    ```

    Replace ALL three `<a href={`/api/staff/${staffId}/calendar/connect`}>` occurrences (connect + two reconnect buttons) with:
    ```tsx
    <Button size="sm" onClick={handleConnect}>
      <Link className="w-4 h-4 mr-2" />
      Connect Google Calendar
    </Button>
    ```
    (adjust text/variant per context — keep existing variants and icons)

    **`server/routes/staff.ts` — accept token query param on connect endpoint:**

    In `GET /:id/calendar/connect`, before calling requireAuth, add a middleware that moves the token query param to the Authorization header if present. OR simpler: modify the connect route to read `req.query.token` as fallback for requireAuth.

    Actually the cleanest way: in the connect route handler, after `requireAuth` passes, use the existing `(req as any).user`. The `requireAuth` middleware needs to accept token from query param as fallback.

    In `server/lib/auth.ts` in the `requireAuth` middleware, add query param fallback:
    ```typescript
    // Check Authorization header first, then query param
    const authHeader = req.headers.authorization;
    const queryToken = req.query.token as string | undefined;
    const token = authHeader?.replace('Bearer ', '') || queryToken;
    ```

    Then validate `token` as before.

    **Remove unused `apiRequest` import** from CalendarTab if it's no longer used after this change.
  </action>
