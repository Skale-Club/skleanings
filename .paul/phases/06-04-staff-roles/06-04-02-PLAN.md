## Goal
Add a `useMe()` hook that fetches `/api/me` and use it in Admin.tsx to conditionally show/hide sidebar sections based on the logged-in user's role.

## Purpose
Staff users (role='staff') should only see Calendar (filtered to their own bookings) and not full admin sections. Admin role sees everything unchanged. Viewer or unknown role falls back to calendar-only.

- `client/src/hooks/useMe.ts` — hook returning `{ role, staffMemberId, isAdmin, isStaff }`
- Admin.tsx updated: fetches role on mount, filters menuItems based on role
- Staff role: sees Dashboard + Calendar only; Calendar pre-filters to their staffMemberId

  <name>Task 1: Create useMe hook</name>
  <files>client/src/hooks/useMe.ts</files>
  <action>
    Create `client/src/hooks/useMe.ts`:

    ```ts
    import { useQuery } from '@tanstack/react-query';
    import { useAdminAuth } from '@/context/AuthContext';

    interface MeData {
      id: string;
      email: string;
      role: 'admin' | 'staff' | 'viewer';
      staffMemberId: number | null;
    }

    export function useMe() {
      const { getAccessToken } = useAdminAuth();

      const { data } = useQuery<MeData>({
        queryKey: ['/api/me'],
        queryFn: async () => {
          const token = await getAccessToken();
          if (!token) throw new Error('No access token');
          const res = await fetch('/api/me', {
            headers: { Authorization: `Bearer ${token}` },
          });
          if (!res.ok) throw new Error('Failed to fetch user profile');
          return res.json();
        },
        staleTime: 5 * 60 * 1000, // 5 min — role doesn't change often
      });

      return {
        role: data?.role ?? 'admin', // default admin while loading (prevents flicker for existing users)
        staffMemberId: data?.staffMemberId ?? null,
        isAdmin: !data || data.role === 'admin', // treat loading as admin (no flicker)
        isStaff: data?.role === 'staff',
      };
    }
    ```

    The default-to-admin-while-loading pattern prevents sidebar flicker for the existing single-admin use case.
  </action>
