<objective>
## Goal
Add GCal busy-time overlay (async gray blocks per staff per day) and click interactions to the calendar — event click opens the booking edit dialog, empty slot click opens the new booking dialog pre-filled with date/time/staff.

## Purpose
The calendar is read-only after Plan 02. This plan makes it actionable: admins can open, edit, or create bookings directly from the calendar without navigating to the Bookings list.

## Output
- GCal busy times fetched async for visible date range, rendered as gray non-interactive background blocks
- Click on booking event → opens booking detail/edit dialog (inline modal, not navigation)
- Click on empty calendar slot → opens new booking dialog pre-filled with date, startTime, staffMemberId (if staff filter active)
</objective>

    1. **New state:** `const [gcalBusy, setGcalBusy] = useState<CalendarEvent[]>([]);`

    2. **Async GCal fetch effect** — runs when `from`, `to`, or `staffList` change:
    ```ts
    useEffect(() => {
      if (!staffList.length) return;
      let cancelled = false;

      async function fetchBusyTimes() {
        const token = await getAccessToken();
        const results: CalendarEvent[] = [];

        // Generate all dates in the visible range
        const dates: string[] = [];
        let d = new Date(from + 'T00:00:00');
        const end = new Date(to + 'T00:00:00');
        while (d <= end) {
          dates.push(format(d, 'yyyy-MM-dd'));
          d = addDays(d, 1);
        }

        // Fetch busy times per staff per date (parallel per staff, sequential per date is fine)
        await Promise.all(
          staffList.map(async (staff) => {
            if (hiddenStaff.has(staff.id)) return;
            await Promise.all(
              dates.map(async (date) => {
                try {
                  const res = await fetch(
                    `/api/staff/${staff.id}/availability?date=${date}`,
                    { headers: { Authorization: `Bearer ${token}` } }
                  );
                  if (!res.ok) return;
                  const data = await res.json() as { busyTimes?: { start: string; end: string }[] };
                  const busyTimes = data.busyTimes ?? [];
                  busyTimes.forEach(({ start, end }) => {
                    results.push({
                      bookingId: -1,
                      title: `${staff.firstName} busy`,
                      start: new Date(`${date}T${start}:00`),
                      end: new Date(`${date}T${end}:00`),
                      status: 'gcal',
                      staffMemberId: staff.id,
                      color: '#E5E7EB',
                      isGcalBusy: true,
                    });
                  });
                } catch {
                  // Silently ignore — GCal overlay is best-effort
                }
              })
            );
          })
        );

        if (!cancelled) setGcalBusy(results);
      }

      fetchBusyTimes();
      return () => { cancelled = true; };
    }, [from, to, staffList, hiddenStaff, getAccessToken]);
    ```

    3. **Combine events:** Replace `events` passed to `<Calendar>` with:
    ```ts
    const allEvents = [...events, ...gcalBusy];
    ```
    Pass `allEvents` to `<Calendar events={allEvents} ...>`.

    Note: The `/api/staff/:id/availability?date=` endpoint already exists from prior phases and returns `{ busyTimes: [] }`. If the endpoint returns a different shape, adapt the destructuring accordingly — read the actual response shape from `server/routes/staff.ts` first.
