<objective>
## Goal
Add a date-range bookings API endpoint, fix the GCal availability perf issue with an in-memory TTL cache, and install react-big-calendar with its CSS — everything Plan 02 needs to build the calendar UI.

## Purpose
The availability engine calls Google Calendar's freeBusy API synchronously on every slot check with no caching — a fresh HTTP request per staff per date on every customer interaction. A simple in-memory cache (10-min TTL) eliminates the bottleneck. The calendar UI also needs a `GET /api/bookings?from=&to=` range query to load multiple days at once.

## Output
- `getBookingsByDateRange(from, to)` on IStorage + DatabaseStorage
- `GET /api/bookings` supports `?from=YYYY-MM-DD&to=YYYY-MM-DD` query params
- In-memory cache on `getStaffBusyTimes()` — 10-min TTL, keyed by `${staffId}:${date}`
- `react-big-calendar` + `date-fns` installed
- react-big-calendar CSS imported in client entry point
</objective>

  <files>server/lib/google-calendar.ts</files>
  <action>
    Add a module-level cache before the `getStaffBusyTimes` function:

    ```ts
    // In-memory cache for GCal busy times — 10-minute TTL
    // Key: `${staffMemberId}:${date}` → { data, expiresAt }
    const busyTimesCache = new Map<string, {
      data: { start: string; end: string }[];
      expiresAt: number;
    }>();

    const BUSY_CACHE_TTL_MS = 10 * 60 * 1000; // 10 minutes
    ```

    Then update `getStaffBusyTimes` to check/populate the cache:
    - At the start of the try block, before the API call:
      ```ts
      const cacheKey = `${staffMemberId}:${date}`;
      const cached = busyTimesCache.get(cacheKey);
      if (cached && cached.expiresAt > Date.now()) {
        return cached.data;
      }
      ```
    - After building `busyIntervals` (before the return):
      ```ts
      const result = busyIntervals.map((interval) => ({ ... })); // existing map
      busyTimesCache.set(cacheKey, { data: result, expiresAt: Date.now() + BUSY_CACHE_TTL_MS });
      return result;
      ```

    Also export a cache-clearing helper for use when a booking is created (prevents stale data for the just-booked date):
    ```ts
    export function clearBusyTimesCache(staffMemberId: number, date: string): void {
      busyTimesCache.delete(`${staffMemberId}:${date}`);
    }
    ```

    Do NOT change the function signature or error handling behavior.
  </action>
