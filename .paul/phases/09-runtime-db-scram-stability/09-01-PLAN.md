<objective>
## Goal
Stabilize production DB authentication and login session behavior so blog autopost and interactive login no longer fail (SCRAM 500s or redirect loop) on cold starts.

## Purpose
Deployment is complete, but production still reports intermittent HTTP 500s with `SASL: SCRAM-SERVER-FINAL-MESSAGE: server signature is missing`, observed in GitHub Actions (`/api/blog/cron/generate`) and in login-linked DB requests. A second issue is now confirmed: login with `skleanings@gmail.com` appears to authenticate and then redirects back to `/login` repeatedly. These symptoms may share runtime/session causes and must be debugged together before feature work continues.

## Output
- Root-cause classification for the SCRAM failure path (URL selection, pool behavior, or transport mismatch)
- Root-cause classification for login redirect loop path (session persistence, auth check, role fetch, or route guard)
- Minimal runtime hardening patch in DB bootstrap path
- Validation evidence for cron-triggered requests and interactive login without redirect looping
</objective>
