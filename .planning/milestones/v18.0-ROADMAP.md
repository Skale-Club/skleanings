# Milestone v18.0: Custom Domain Routing

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 61–62
**Total Plans:** 5

## Overview

Tenant admins can register their own custom domain (e.g. `agendar.minhalimpeza.com`) instead of being limited to `*.xkedule.com` subdomains. DNS verification via TXT record at `_xkedule.<hostname>`. Caddy on-demand TLS handles certificate issuance automatically. `resolveTenantMiddleware` only accepts verified domains for non-primary entries.

## Phases

**Goal**: Tenants can register/verify/list/remove custom domains via authenticated API; resolveTenantMiddleware blocks unverified non-primary hostnames

## Milestone Summary

**Files changed:** ~10 files

**Key decisions:**
- LRU cache invariant: "if it's in the cache, it's safe to serve" — only primary or verified domains reach `hostnameCache.set`
- verificationToken returned ONCE on POST /domains create — never exposed via GET (one-time-secret pattern)
- POST /domains rejects *.xkedule.com hostnames to prevent system subdomain hijacking
- Both 404 branches in middleware use identical "Unknown tenant" message (anti-enumeration)
- Frontend keeps verificationToken in component state only (not localStorage) — user can re-fetch from row Verify button if dialog closed
