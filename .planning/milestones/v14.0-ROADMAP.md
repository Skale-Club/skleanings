# Milestone v14.0: Billing Hardening

**Status:** ✅ SHIPPED 2026-05-14
**Phases:** 53–54
**Total Plans:** 4

## Overview

The billing lifecycle is fully automated — tenants receive Resend email warnings before their trial ends and when their subscription lapses, can view invoice history directly in /admin/billing, and the signup endpoint is rate-limited against abuse.

## Phases

### Phase 53: Billing Email Notifications + Signup Rate Limit

**Goal**: The billing lifecycle sends automated email warnings to tenants at critical subscription events, and the public signup endpoint is protected against abuse via IP-based rate limiting
**Depends on**: Phase 52
**Plans**: 2 plans

Plans:

- [x] 53-01: trial_will_end + past_due Resend email sends in billingWebhookHandler (DatabaseStorage.forTenant lookup, fire-and-forget)
- [x] 53-02: express-rate-limit on POST /api/auth/signup (5/hr per IP, 429 + Retry-After)

### Phase 54: Invoice History

**Goal**: Tenant admins can view their complete invoice history directly within the billing page — the last 10 Stripe invoices are fetched server-side and displayed with status and download links
**Depends on**: Phase 53
**Plans**: 2 plans

Plans:

- [x] 54-01: GET /api/billing/invoices (requireAdmin, stripe.invoices.list limit 10, maps to { id, date, amount, currency, status, invoiceUrl })
- [x] 54-02: Invoice History card in BillingPage.tsx (useQuery, Skeleton loading, empty state, Table with date/amount/status Badge/Download anchor)

## Milestone Summary

**Files changed:** ~15 files across 2 phases

**Key decisions:**
- Billing email sends are non-fatal (inner try/catch) — Stripe webhook always gets 200 even if Resend fails
- DatabaseStorage.forTenant(subRow.tenantId) used in webhook handler for tenant-scoped email context (no res.locals.storage available in webhook)
- GET /api/billing/invoices returns `{ invoices: [] }` (not 404) when tenant has no stripeCustomerId — matches existing GET /status pattern
- useQuery (not useState+useEffect) used for invoice fetch per BH-06 requirement
- signupRateLimit uses standardHeaders: true for RFC 6585 RateLimit-* headers including Retry-After
