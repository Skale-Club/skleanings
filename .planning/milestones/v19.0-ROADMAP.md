# Milestone v19.0: Stripe Connect Onboarding

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 63–64
**Total Plans:** 5

## Overview

Tenants can connect their own Stripe Express account via Stripe-hosted onboarding. Connection state and capability flags (charges_enabled, payouts_enabled, details_submitted) persist in `tenant_stripe_accounts` and stay in sync via webhooks. UI surfaces connection status to both tenant admin (/admin/payments) and super-admin (Tenants table Connect column).

**Note:** Payment flow integration — actually routing customer payments through tenant accounts with `application_fee_amount` — is intentionally deferred to v20.0.

## Milestone Summary

**Files changed:** ~10 files

**Key decisions:**
- Stripe Express (not Standard/Custom) — minimizes compliance burden, Stripe-hosted onboarding
- Persist DB row BEFORE generating AccountLink — prevents orphaned Stripe accounts
- LEFT JOIN in super-admin tenants query — tenants without Connect rows still appear with `connected: false`
- Webhook uses `event.account` (top-level Connect event field) for deauthorized events, not `event.data.object` (which Stripe types as `Application`)
- Hard-delete on deauthorize — irreversible action, fresh re-onboarding mints new Express account
