# Milestone v13.0: Self-Serve Signup

**Status:** ✅ SHIPPED 2026-05-14
**Phases:** 51–52
**Total Plans:** 4

## Overview

Any business can sign up independently via a public page — no super-admin required to onboard a new tenant. Signup atomically provisions the full tenant stack and starts a 14-day Stripe free trial. The /admin/billing page shows trial status and guides admins to add a payment method.

## Phases

### Phase 51: Self-Serve Signup Backend

**Goal**: Any business can sign up without super-admin involvement — a single POST endpoint atomically provisions the full tenant stack (tenant, domain, admin user, company settings, Stripe customer, trial subscription) and the Stripe webhook keeps trial status in sync
**Depends on**: Phase 50
**Plans**: 2 plans

Plans:

- [x] 51-01: signupTenant() IStorage method + POST /api/auth/signup route (atomic provision + Stripe 14-day trial + session setup)
- [x] 51-02: billingWebhookHandler: customer.subscription.trial_will_end event + ROADMAP.md plan count update

### Phase 52: Self-Serve Signup Frontend

**Goal**: A business owner can discover the platform, fill in their details on a public signup page, and land in their own admin panel — the /admin/billing page communicates trial status and guides them to add a payment method before the trial ends
**Depends on**: Phase 51
**Plans**: 2 plans

Plans:

- [x] 52-01: Signup.tsx public page (5 fields, live subdomain preview, inline validation, cross-subdomain redirect) + App.tsx route
- [x] 52-02: BillingPage.tsx trial UI — Trial badge, days-remaining countdown, Add Payment Method CTA for trialing/past_due
