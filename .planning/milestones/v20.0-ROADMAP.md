# Milestone v20.0: Connect Payment Routing

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 65–66
**Total Plans:** 5

## Overview

Customer booking payments are routed through the tenant's connected Stripe Express account using the platform Stripe key + `Stripe-Account` header with an `application_fee_amount` skimmed by the platform. Legacy per-tenant Stripe API key flow continues to work for tenants not yet on Connect. `/admin/payments` shows recent payments with platform fee + net-to-tenant breakdown.

## Milestone Summary

**Files changed:** ~10 files

**Key decisions:**
- Discriminated union for Stripe context (connect/legacy/connect-incomplete/none) makes explicit handling required
- Dual-secret webhook verification — two-pass: Connect secret first, legacy fallback, never parse unverified payload
- Connect events use stripe.checkout.sessions.retrieve(sessionId, { expand: ['payment_intent'] }, { stripeAccount: event.account })
- application_fee_amount calculated as Math.max(1, Math.floor(total * percent / 100)) — minimum 1 cent for Stripe
- amountTotal returned as integer cents (totalPrice × 100) for uniform client-side division
- Refresh button invalidates BOTH /api/admin/stripe/status AND /api/admin/payments/recent
