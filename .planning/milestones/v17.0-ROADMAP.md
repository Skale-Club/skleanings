# Milestone v17.0: Plan Tiers

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 59–60
**Total Plans:** 5

## Overview

Tenants can be assigned to one of three Stripe plan tiers (Basic, Pro, Enterprise) with different price IDs and feature gates. Super-admin sets the plan; tenant admin sees their tier and feature list; backend enforces feature limits via a single `tenantHasFeature()` helper.

## Milestone Summary

**Files changed:** ~10 files

**Key decisions:**
- planTier kept as TEXT (not pgEnum) matching existing status column pattern
- Helpers read process.env at call-time (not module-load) for testability
- Conditional spread in webhook leaves planTier untouched on unrecognized priceId
- Optimistic DB update in PATCH + webhook reconciliation (mirrors POST /subscribe)
- Select renders em-dash for tenants without active subscriptions (avoids PATCH 404)
- Silent React Query invalidation on tier change (no success toast — design choice for MVP)
