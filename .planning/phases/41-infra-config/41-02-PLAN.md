Create `.github/workflows/deploy.yml` with this exact content. The on: block must contain ONLY workflow_dispatch — no push, no schedule trigger whatsoever (MT-16 constraint).

File content:

    name: Deploy to Hetzner

    on:
      workflow_dispatch:
        inputs:
          reason:
            description: 'Reason for this deployment'
            required: false
            default: 'Manual deploy'

    jobs:
      deploy:
        name: SSH Deploy to Hetzner CX23
        runs-on: ubuntu-latest
        timeout-minutes: 10
        steps:
          - name: Deploy via SSH
            uses: appleboy/ssh-action@v1
            with:
              host: ${{ secrets.HETZNER_HOST }}
              username: ${{ secrets.HETZNER_USER }}
              key: ${{ secrets.HETZNER_SSH_KEY }}
              port: 22
              script: |
                set -e
                cd /var/www/skleanings
                git pull origin main
                npm ci
                npm run build
                sudo systemctl restart skleanings
                echo "Deploy complete at $(date -u)"

Add a comment block at the top of the file (before `name:`) explaining:
- TRIGGER: Manual only (workflow_dispatch). This workflow NEVER runs automatically on push.
- Required GitHub Secrets: HETZNER_SSH_KEY, HETZNER_HOST, HETZNER_USER
- See infra/README.md for setup instructions

Critical implementation notes:
- `npm ci` without --omit=dev is required — devDependencies (vite, esbuild) are needed for npm run build
- `set -e` causes fail-fast if any command exits non-zero — prevents partial deploys
- `sudo systemctl restart skleanings` requires passwordless sudo for the deploy user (documented in README)
- Unit name `skleanings` must match the service installed from infra/app.service

Create `infra/README.md` as a complete Hetzner CX23 server setup guide. The file must be a standalone reference — an operator unfamiliar with the codebase should be able to follow it from a fresh server to a working deployment.

Structure the README with these 8 sections in order, using ## headings. Use blank lines between sections instead of horizontal rules. Each section must include all bash commands and explanations from the research document.

Section 1 "Server Overview":
  - Server: Hetzner CX23 (2 vCPU, 4 GB RAM, Ubuntu 22.04 LTS)
  - Domain: *.xkedule.com (wildcard) + custom tenant domains
  - Reverse Proxy: Caddy (custom xcaddy build with Cloudflare DNS module)
  - Process Manager: systemd (direct Node.js, no PM2)
  - Node Version: 20 LTS

Section 2 "Step 1: Initial Server Setup":
  - Create 'skleanings' user, add to sudo group
  - Configure passwordless sudo: echo 'skleanings ALL=(ALL) NOPASSWD: /bin/systemctl restart skleanings' >> /etc/sudoers.d/skleanings
  - ufw allow 22, 80, 443 and ufw enable

Section 3 "Step 2: Install Node.js 20 LTS":
  - NodeSource apt repo install for Node 20 (not apt install nodejs — gives old version)
  - Include the full NodeSource install commands from the research (NODE_MAJOR=20, curl gpgkey, echo deb, apt-get install nodejs)
  - Note: NOT nvm — systemd ExecStart needs /usr/bin/node absolute path; nvm installs to ~/.nvm which systemd cannot find
  - Verify with: node --version (expect v20.x.x) and which node (expect /usr/bin/node)

Section 4 "Step 3: Install xcaddy and Build Caddy":
  - CRITICAL callout: Standard 'apt install caddy' cannot issue wildcard TLS certificates
  - Explain: *.xkedule.com requires DNS-01 challenge which requires caddy-dns/cloudflare plugin built via xcaddy
  - Full xcaddy install commands from research (debian-keyring, curl gpgkey, tee sources list, apt install xcaddy)
  - Build command: xcaddy build --with github.com/caddy-dns/cloudflare
  - Replace binary: cp caddy /usr/bin/caddy && caddy version

Section 5 "Step 4: Configure Caddy":
  - mkdir -p /etc/caddy
  - cp infra/Caddyfile /etc/caddy/Caddyfile
  - Create /etc/caddy/caddy.env with CLOUDFLARE_API_TOKEN (chmod 600)
  - Edit Caddy systemd unit to add EnvironmentFile=/etc/caddy/caddy.env under [Service]
  - systemctl daemon-reload, enable caddy, start caddy, status caddy
  - Cloudflare API Token requirements: Zone > DNS > Edit permission
  - How to add custom tenant domain: new site block in Caddyfile + caddy reload command

Section 6 "Step 5: Clone Repository and Configure Environment":
  - mkdir -p /var/www/skleanings + chown skleanings:skleanings
  - git clone as skleanings user, npm ci, npm run build
  - Create /etc/skleanings/.env (mkdir, touch, chmod 600, chown skleanings)
  - Show the env file format: bare KEY=value, no 'export', no shell quoting
  - List all required env vars: NODE_ENV, PORT, DATABASE_URL, SESSION_SECRET, ADMIN_EMAIL, ADMIN_PASSWORD_HASH, SUPER_ADMIN_EMAIL, SUPER_ADMIN_PASSWORD_HASH, CLOUDFLARE_API_TOKEN
  - WARNING: 'export KEY=value' or quoted values causes systemd "Failed to load environment files" error
  - Bcrypt hash generation: node -e "const bcrypt = require('bcrypt'); bcrypt.hash('yourpassword', 10).then(h => console.log(h));"

Section 7 "Step 6: Install systemd Service":
  - cp infra/app.service /etc/systemd/system/skleanings.service
  - systemctl daemon-reload, enable skleanings, start skleanings, status skleanings
  - Log viewing: journalctl -u skleanings -f

Section 8 "Step 7: Cloudflare DNS Configuration":
  - Add wildcard A record: Name=*, value=Hetzner IP, Proxy=Proxied (orange cloud)
  - Add root A record: Name=@, value=Hetzner IP, Proxy=Proxied (orange cloud)
  - SSL/TLS mode: Full (strict) — Caddy manages valid Let's Encrypt certs
  - Note: DNS-01 challenge uses TXT records, unaffected by Cloudflare proxy status

Section 9 "Step 8: Configure GitHub Secrets for Deploy Workflow":
  - Location: Settings > Secrets and variables > Actions in the GitHub repo
  - Three secrets table: HETZNER_SSH_KEY, HETZNER_HOST, HETZNER_USER with descriptions
  - Ed25519 keypair generation: ssh-keygen -t ed25519 -C "github-deploy" -f ~/.ssh/hetzner_deploy
  - Where to put public key (server authorized_keys) and private key (GitHub secret)
  - How to trigger: Actions > Deploy to Hetzner > Run workflow

Section 10 "Troubleshooting":
  - Table with: Symptom | Likely cause | Fix
  - Row 1: "Failed to load environment files" | 'export' prefix or quotes in .env | Rewrite with bare KEY=value
  - Row 2: Caddy "unrecognized module" or "dns provider not found" | Standard apt caddy binary | Rebuild with xcaddy (Step 3)
  - Row 3: Wildcard cert issuance fails | CLOUDFLARE_API_TOKEN missing/wrong permissions | Verify Zone > DNS > Edit scope
  - Row 4: systemctl start skleanings exits immediately | Wrong node path | Run 'which node' must be /usr/bin/node; check journalctl -u skleanings
  - Row 5: Deploy SSH auth fails | Wrong key or not in authorized_keys | Verify Ed25519 key pair
  - Row 6: Build fails "vite: command not found" | npm ci --production used | Use plain npm ci (no --omit=dev)
