```caddyfile
# /etc/caddy/Caddyfile
# Caddy reverse proxy configuration for Skleanings multi-tenant platform
#
# IMPORTANT: This file requires a custom Caddy binary built with the Cloudflare DNS module.
# Standard 'apt install caddy' cannot issue wildcard certificates.
# Build with: xcaddy build --with github.com/caddy-dns/cloudflare
# Then: sudo cp caddy /usr/bin/caddy
#
# See infra/README.md for full setup instructions.

{
    email admin@xkedule.com
}

# Wildcard subdomain block — covers *.xkedule.com
# DNS-01 challenge via Cloudflare API (works with Cloudflare proxy enabled)
# Requires CLOUDFLARE_API_TOKEN env var with Zone > DNS > Edit permission
*.xkedule.com {
    tls {
        dns cloudflare {env.CLOUDFLARE_API_TOKEN}
    }
    reverse_proxy localhost:5000
}

# Custom tenant domains — add one block per custom domain
# Caddy issues HTTP-01 certificates automatically (no DNS plugin needed for non-wildcard domains)
# Requires port 80 accessible from the internet for HTTP-01 challenge
# After adding a new block: sudo caddy reload --config /etc/caddy/Caddyfile
#
# Example:
# cleaning.tenantcustomdomain.com {
#     reverse_proxy localhost:5000
# }
```

Notes:
- The `{env.CLOUDFLARE_API_TOKEN}` syntax reads the token from the environment at Caddy startup — do NOT hardcode the token in this file
- Caddy does NOT need to know individual tenant subdomains — all *.xkedule.com traffic proxies to Node which resolves the tenant (per Phase 40 resolveTenantMiddleware)
- Custom domains get HTTP-01 certs automatically on first request; no DNS plugin needed for those

```ini
# /etc/systemd/system/skleanings.service
# systemd unit for the Skleanings multi-tenant booking platform
#
# Installation:
#   sudo cp infra/app.service /etc/systemd/system/skleanings.service
#   sudo systemctl daemon-reload
#   sudo systemctl enable skleanings
#   sudo systemctl start skleanings
#
# Logs: journalctl -u skleanings -f
# Status: systemctl status skleanings
#
# See infra/README.md for full setup instructions including env file format.

[Unit]
Description=Skleanings Multi-Tenant Booking Platform
After=network.target

[Service]
Type=simple
User=skleanings
Group=skleanings
WorkingDirectory=/var/www/skleanings
EnvironmentFile=/etc/skleanings/.env
ExecStart=/usr/bin/node dist/index.cjs
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=skleanings

[Install]
WantedBy=multi-user.target
```

Critical notes:
- `Type=simple` is correct — Node does not fork. Using `Type=forking` would cause systemd to think the process failed to start.
- `EnvironmentFile=/etc/skleanings/.env` loads env vars from the server-side env file (bare KEY=value format, never committed to git).
- `ExecStart=/usr/bin/node dist/index.cjs` uses the absolute path — Node installed via NodeSource apt is at /usr/bin/node. Do NOT use `node` without the full path (systemd minimal PATH may not include it).
- The unit name `skleanings` must match what deploy.yml uses in `sudo systemctl restart skleanings`.
