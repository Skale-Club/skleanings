**Client-side validation in handleSubmit** (runs before any fetch):
- All five fields required — set `errors.{field}` if empty
- `slug` regex: `/^[a-z0-9-]+$/` and min length 2 — error: "Only lowercase letters, numbers, and hyphens"
- `password` min 8 chars — error: "Password must be at least 8 characters"
- `confirmPassword !== password` — error: "Passwords do not match" on `confirmPassword`
- If any errors object has keys, return early without calling fetch

**API call:**
```typescript
const res = await fetch('/api/auth/signup', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ companyName, slug, email, password }),
});
```

**Response handling:**
- 201: `const { adminUrl } = await res.json(); window.location.href = adminUrl;`
- 409: `const { field, message } = await res.json(); setErrors({ [field]: message });`
- Other non-ok: set a generic `errors.form` message
