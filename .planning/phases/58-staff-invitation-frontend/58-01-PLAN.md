Create `client/src/pages/AcceptInvite.tsx` as a default-exported React function component. Structure:

1. **Imports**: `useState`, `useEffect` from `react`; `Card, CardContent, CardHeader, CardTitle, CardDescription` from `@/components/ui/card`; `Input` from `@/components/ui/input`; `Button` from `@/components/ui/button`; `Label` from `@/components/ui/label`. No router hook required — read the token via `new URLSearchParams(window.location.search).get('token')` (matches the VerifyEmail.tsx pattern).

2. **State** (controlled, mirroring Signup.tsx):
   - `status`: `'loading' | 'invalid' | 'ready' | 'submitting'`
   - `invite`: `{ email: string; companyName: string; role: 'staff' | 'admin' } | null`
   - `name`, `password`, `confirmPassword` strings
   - `errors`: `Record<string, string>`
   - `clearFieldError(field)` helper identical to Signup.tsx

3. **On mount** (`useEffect`, runs once):
   - Read `token` from `URLSearchParams`. If missing → `setStatus('invalid')`, return.
   - `fetch(`/api/auth/validate-invite?token=${encodeURIComponent(token)}`)`.
   - If `res.ok` → parse JSON → set `invite` from `{ email, companyName, role }` → `setStatus('ready')`.
   - If `res.status === 410` or any non-ok → `setStatus('invalid')`.
   - Wrap in try/catch; network errors also → `setStatus('invalid')`.

4. **Render branches**:
   - `status === 'loading'`: centered spinner (copy the spinner div from Signup.tsx `authLoading` branch — `animate-spin rounded-full h-8 w-8 border-b-2 border-primary` inside the gradient background wrapper).
   - `status === 'invalid'`: error card mirroring VerifyEmail.tsx — title "Invitation expired or already used", description "This invitation link is no longer valid. Ask the person who invited you to send a new one." No form. No resend button.
   - `status === 'ready' | 'submitting'`: setup form (see below).

5. **Form** (only when invite is loaded):
   - Header: `CardTitle` = "Accept your invitation", `CardDescription` = `You're invited to join <strong>{invite.companyName}</strong> as <strong>{invite.role}</strong>`. Render with JSX `<strong>` tags, not raw HTML.
   - Read-only Email field: `<Input id="email" type="email" value={invite.email} disabled />` with Label "Email".
   - Name input: required, min length 2. `value={name}` controlled.
   - Password input: type="password", min length 8. `value={password}` controlled.
   - Confirm Password input: type="password", must match `password`. `value={confirmPassword}` controlled.
   - Each field renders its `errors[field]` below in `text-sm text-destructive mt-1` exactly like Signup.tsx.
   - Submit Button: `className="w-full bg-[#FFFF01] text-black font-bold rounded-full hover:bg-yellow-300"` — disabled while `status === 'submitting'`. Label: `'Accept Invitation'` or `'Accepting...'`.

6. **Submit handler** (`handleSubmit(e)`):
   - `e.preventDefault()`. Build `newErrors`:
     - if `name.trim().length < 2` → `newErrors.name = 'Name must be at least 2 characters'`
     - if `password.length < 8` → `newErrors.password = 'Password must be at least 8 characters'`
     - if `confirmPassword !== password` → `newErrors.confirmPassword = 'Passwords do not match'`
   - If any errors → `setErrors(newErrors)`, return.
   - `setStatus('submitting')`. POST to `/api/auth/accept-invite` with JSON body `{ token, name: name.trim(), password }`. Headers `Content-Type: application/json`. `credentials: 'include'`.
   - On `res.status === 201`: parse `{ adminUrl }`, `window.location.href = adminUrl`. Do NOT setStatus afterward (we are navigating away).
   - On `res.status === 410`: `setStatus('invalid')`.
   - On any other non-ok: parse `{ message }` if JSON, set `errors.form = message ?? 'Could not accept invitation. Please try again.'`, revert `setStatus('ready')`.
   - Network catch: same as the `else` branch above with a network-specific message; revert to `'ready'`.

7. **Layout shell** (all branches): same outer wrapper as Signup.tsx — `min-h-screen flex items-center justify-center bg-gradient-to-br from-slate-50 to-slate-100 p-4` with `w-full max-w-md` inner div.

Do NOT add anything related to login, signup links, or resend — this is a one-shot acceptance flow. Do NOT introduce a new hook file; keep state local. Do NOT import `useAdminTenantAuth` — accepting an invite establishes its own session server-side, and the resulting redirect goes to a different subdomain.
