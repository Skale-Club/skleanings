API surface (mounted at /api/admin/domains, requireAdmin + tenant-resolved):

```
GET /api/admin/domains
  → 200 { domains: Array<{
      id: number;
      hostname: string;
      isPrimary: boolean;
      verified: boolean;
      verifiedAt: string | null;     // ISO timestamp or null
      createdAt: string;
    }> }
  Note: verificationToken is NEVER returned by GET — only by POST create.

POST /api/admin/domains    body: { hostname: string }
  → 201 {
      id: number;
      hostname: string;
      verificationToken: string;       // 32-byte hex, shown ONCE
      instructions: {
        recordType: "TXT";
        recordName: string;            // _xkedule.<hostname>
        recordValue: string;           // same as verificationToken
        message: string;
      }
    }
  → 400 { message }   (invalid hostname / reserved xkedule.com subdomain)
  → 409 { message: "Domain already registered" }

POST /api/admin/domains/:id/verify
  → 200 { verified: true, alreadyVerified?: boolean }
  → 400 { verified: false, message: string }   (TXT mismatch / DNS error)
  → 404 { message: "Domain not found" }

DELETE /api/admin/domains/:id
  → 200 { message }
  → 409 { message }   (cannot remove primary)
  → 404 { message }
```

   ```ts
     verified: boolean;
     verifiedAt: string | null;

   };
   type AddDomainResponse = {
     id: number;
     hostname: string;
     verificationToken: string;
     instructions: { recordType: string; recordName: string; recordValue: string; message: string };
   };
   ```

2. **List query**: `useQuery<{ domains: DomainItem[] }>({ queryKey: ['/api/admin/domains'] })` — relies on the default queryFn already configured in `client/src/lib/queryClient.ts`. Show a Loader2 spinner while loading.

3. **Render** (Card with CardHeader/CardContent):
   - Header row: title "Domains" (Outfit font, text-xl font-semibold) + "+ Add Custom Domain" button (Brand Yellow `#FFFF01`, rounded-full, font-bold text-black) that opens the AddDomainDialog.
   - shadcn Table with columns: Hostname (font-mono) | Status | Actions.
   - Status cell renders ONE badge based on priority: `isPrimary` → purple "Primary"; else `verified` → green "Verified"; else yellow "Pending Verification".
   - Actions cell: "Verify" button (only when !isPrimary && !verified) and "Remove" button (only when !isPrimary). Use shadcn Button `variant="ghost" size="sm"`. Remove uses `text-red-600 hover:text-red-800`.

4. **AddDomainDialog** (internal sub-component using shadcn Dialog):
   - State: `hostname: string`, `result: AddDomainResponse | null`, `error: string | null`.
   - When `result === null`: render Input + "Add" button. On submit, POST to `/api/admin/domains` with `{ hostname: hostname.trim().toLowerCase() }` and `credentials: 'include'`, `Content-Type: application/json`. On 409, surface `data.message`. On 201, set `result` and invalidate `['/api/admin/domains']`.
   - When `result !== null`: render a DNS instructions panel showing the exact TXT record. Use a styled box (bg-slate-50 border rounded-md p-4 font-mono text-sm) with three labeled rows: "Record type: TXT", "Name: {result.instructions.recordName}", "Value: {result.verificationToken}". Add a Copy button next to the value (uses `navigator.clipboard.writeText`). Below the box, render a `Verify` button (Brand Yellow) that calls `POST /api/admin/domains/${result.id}/verify`. The token MUST stay in component state until the user verifies OR closes the dialog (closing resets `result` to null AND keeps the row visible in the list — user can verify later from the table row).

5. **Verify mutation** (also reusable from the table-row Verify button):
   - `POST /api/admin/domains/${id}/verify` with `credentials: 'include'`.
   - On 200: toast `{ title: 'Domain verified!', description: hostname }`, invalidate `['/api/admin/domains']`, close dialog if open.
   - On non-200: read body `{ message }` and toast `{ title: 'Verification failed', description: message, variant: 'destructive' }`.

6. **Remove flow**:
   - Use shadcn `AlertDialog` (NOT `window.confirm`) — title "Remove domain?", description includes the hostname, destructive `AlertDialogAction` labeled "Remove" with `bg-red-600 text-white`.
   - On confirm: `DELETE /api/admin/domains/${id}` with `credentials: 'include'`. On success: invalidate list + toast `{ title: 'Domain removed' }`. On 409: toast `{ title: 'Cannot remove', description: 'Primary domain cannot be removed', variant: 'destructive' }`.

7. **Empty state**: when `domains.length === 0`, render a centered placeholder "No domains yet. Add your first custom domain to get started." with the same Add button.

Use `useToast` from `@/hooks/use-toast`. Use `queryClient` from `@/lib/queryClient` for `invalidateQueries`. DO NOT roll a custom auth helper — session cookie via `credentials: 'include'` is sufficient (per existing tenant-admin pattern).

Avoid: keeping the verificationToken in localStorage (security — it's in component state only); blocking the dialog while showing the DNS panel (user must be able to close & come back to verify later); calling `verify` automatically on dialog open (must be explicit click).
