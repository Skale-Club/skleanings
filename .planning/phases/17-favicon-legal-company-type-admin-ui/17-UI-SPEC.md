## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline indicator gaps |
| sm | 8px | Compact element spacing (`gap-2`, `space-x-2`) |
| md | 16px | Default field spacing (`gap-4`, `space-y-4`) |
| lg | 24px | Section padding, card inner padding (`p-6`) |
| xl | 32px | Layout gaps between cards (`space-y-8` within the main section) |
| 2xl | 48px | Public page section vertical padding (`py-12`) |
| 3xl | 64px | Public page hero vertical padding (`py-16`) |

Exceptions:
- Favicon preview thumbnail: 48x48px (touch-safe minimum for the clickable upload zone)
- `<Separator>` between Legal & Branding card sub-sections: 1px divider, no spacing token (uses default `my-0`)
- Textarea min-height for legal content: `min-h-[400px]` (not a spacing token — field height exception)

Source: existing `CompanySettingsSection.tsx` patterns (`p-6`, `space-y-6`, `gap-4`); CONTEXT.md D-11

## Interaction Contracts

### Favicon Upload

| State | Visual |
|-------|--------|
| No favicon set | Placeholder area: dashed border `border-2 border-dashed border-muted-foreground/30 rounded-lg p-4`, centered upload icon (`Image` lucide, `w-12 h-12 text-muted-foreground`) |
| Favicon set | 48×48px `<img>` with `object-contain rounded` + filename or URL truncated below |
| Uploading | Button shows `<Loader2 className="h-4 w-4 animate-spin" />` + "Uploading..." text, disabled state |
| Upload success | Auto-save fires immediately (no debounce — direct `saveSettings({ faviconUrl: objectPath })`), toast "Asset uploaded and saved" |
| Upload error | Toast destructive: "Upload failed — [error message]" |

Trigger: hidden `<input type="file" accept=".png,.ico,.svg,.jpg,.jpeg">` activated by Button click.

### Service Delivery Model RadioGroup

| State | Visual |
|-------|--------|
| Default (no saved value) | `at-customer` pre-selected |
| Option selected | RadioGroupItem filled, Label title bold, subtitle muted |
| Change | `updateField('serviceDeliveryModel', value)` — triggers 800ms debounce auto-save |
| Saving | Global auto-save indicator (Loader2 + "Saving..." in page header) |

No separate confirm step — radio change is immediately staged for auto-save.

### Legal Textarea

| State | Visual |
|-------|--------|
| Empty | Placeholder text visible: "Paste your privacy policy content here..." |
| Typing/pasting | `updateField('privacyPolicyContent', value)` on `onChange`, 800ms debounce auto-save |
| Content present | Monospace text displayed in full height field |
| Saving | Global auto-save indicator |

### Public Legal Pages

| State | Visual |
|-------|--------|
| Loading | No explicit loading spinner — page renders hero header immediately; content zone shows nothing while query resolves (query is fast, same endpoint as rest of site) |
| Content present | `prose prose-gray max-w-none` div with raw HTML rendered |
| Content empty / null / whitespace-only | `LegalEmptyState` centered card (see layout above) |

## Accessibility Contract

- All form fields have explicit `<Label htmlFor="...">` with matching `id` on the control
- RadioGroup items: each `<RadioGroupItem>` has `id={`sdm-${value}`}` and matching `<Label htmlFor={`sdm-${value}`}>`
- Favicon upload `<input type="file">` is hidden but accessible: triggered via `<Button>` with `aria-label="Upload favicon"`
- Textarea fields: `aria-describedby` pointing to helper text `<p>` elements (use `id="privacy-helper"` / `id="terms-helper"`)
- Auto-save status region: wrap the saving/saved indicator in `role="status" aria-live="polite"` so screen readers announce saves
- Public legal pages: `<h1>` is the first heading in document order; prose content must not introduce a second `<h1>`
- Empty state text: plain `<p>` elements, no ARIA roles needed
