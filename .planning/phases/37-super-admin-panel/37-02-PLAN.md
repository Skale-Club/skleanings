  <action>
    Create server/routes/super-admin.ts with ALL /api/super-admin/* handlers. Implement exactly
    as specified in 37-RESEARCH.md Patterns 1–4. Key requirements:

    **Imports needed:**
    - `bcrypt` from "bcrypt"
    - `Router, Request, Response, NextFunction` from "express"
    - `db` from "../db"
    - `ensureDatabaseReady` from "../db"
    - `bookings, contacts, services, staffMembers` from "@shared/schema"
    - `count, eq` from "drizzle-orm"
    - `collectRuntimeEnvDiagnostics` from "../lib/runtime-env"
    - `getRecentErrors` from "../lib/error-log"
    - `storage` from "../storage"
    - `fs` and `path` as Node built-ins (use `import fs from "fs"` and `import path from "path"`)

    **requireSuperAdmin middleware (export it):**
    - Check `req.session.superAdmin?.authenticated === true`
    - Return `res.status(403).json({ message: "Super-admin access required" })` if not authenticated
    - Return `next()` if authenticated

    **POST /login:**
    - Destructure `{ email, password }` from `req.body`
    - Get `expectedEmail = process.env.SUPER_ADMIN_EMAIL` and `expectedHash = process.env.SUPER_ADMIN_PASSWORD_HASH`
    - If any is missing/falsy: return 403 `{ message: "Invalid credentials" }`
    - CRITICAL timing-safe check (per research Pitfall 4): use `Promise.all([...])` so bcrypt.compare
      ALWAYS runs regardless of email match:
      ```typescript
      const [emailMatch, hashMatch] = await Promise.all([
        Promise.resolve(email.toLowerCase() === expectedEmail.toLowerCase()),
        bcrypt.compare(password, expectedHash),
      ]);
      ```
    - If either false: return 403 `{ message: "Invalid credentials" }`
    - Set `req.session.superAdmin = { authenticated: true }` and return `res.json({ ok: true })`

    **POST /logout:**
    - Call `req.session.destroy(() => res.json({ ok: true }))`

    **GET /me:**
    - No requireSuperAdmin middleware (public check)
    - If `req.session.superAdmin?.authenticated === true` → return `res.json({ authenticated: true })`
    - Else → return `res.status(403).json({ authenticated: false })`

    **GET /stats (requireSuperAdmin):**
    - Use `Promise.all` with 4 parallel `db.select({ count: count() })` queries:
      - bookings → total count (no filter)
      - contacts → total count (no filter)
      - services → count where `eq(services.isArchived, false)`
      - staffMembers → count where `eq(staffMembers.isActive, true)`
    - Return `{ totalBookings, totalContacts, totalServices, totalStaff, serverUptimeSeconds: Math.floor(process.uptime()) }`
    - Wrap in try/catch, return 500 on error

    **GET /health (requireSuperAdmin):**
    - Check DB: try `await ensureDatabaseReady()` → `dbConnected = true`, catch → `dbConnected = false`
    - Count migrations: `fs.readdirSync(path.join(process.cwd(), "supabase", "migrations")).filter(f => f.endsWith(".sql")).length`
      wrapped in try/catch (default 0 if directory missing)
    - Call `collectRuntimeEnvDiagnostics()` to get `{ errors, warnings }`
    - Return `{ dbConnected, migrationCount, envErrors: errors, envWarnings: warnings }`

    **GET /company-settings (requireSuperAdmin):**
    - Return `await storage.getCompanySettings()`
    - Wrap in try/catch, return 500 on error

    **PATCH /company-settings (requireSuperAdmin):**
    - Call `await storage.updateCompanySettings(req.body)` and return updated result
    - Wrap in try/catch, return 500 on error

    **GET /error-logs (requireSuperAdmin):**
    - Return `res.json(getRecentErrors())`

    **Export at bottom:**
    ```typescript
    export { router as superAdminRouter };
    ```
