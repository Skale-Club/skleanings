    **Step A — Create server/lib/error-log.ts:**

    Implement the ring buffer module exactly as specified in the research (Pattern 2). The module has:
    - `ErrorEntry` interface with `timestamp: string` (ISO 8601), `message: string`, `stack?: string`
    - Module-level `errorLog: ErrorEntry[]` array, `MAX_ENTRIES = 50`
    - `appendError(entry: ErrorEntry): void` — pushes and shifts when over MAX_ENTRIES
    - `getRecentErrors(): ErrorEntry[]` — returns `[...errorLog].reverse()` (most recent first)
    - `patchConsoleError(): void` — idempotent (guarded by `let patched = false`), monkey-patches
      `console.error` to call the original AND appendError. Extracts message from all args (joining
      non-Error args as String(a), using .message for Error args). Extracts stack from first Error arg.

    Use the exact implementation from 37-RESEARCH.md Pattern 2 — do not deviate.
