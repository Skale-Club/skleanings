       - On success: invalidate ['/api/super-admin/me'] query
       - Returns useMutation result (mutate, isPending, isError, error)

    3. `useSuperAdminLogout()` — POST /api/super-admin/logout mutation
       - useMutation: POST /api/super-admin/logout
       - On success: invalidate ['/api/super-admin/me']

    4. `useSuperAdminStats()` — GET /api/super-admin/stats
       - useQuery: queryKey ['/api/super-admin/stats']
       - enabled: only when authenticated (pass `enabled: boolean` parameter)

    5. `useSuperAdminHealth()` — GET /api/super-admin/health
       - useQuery: queryKey ['/api/super-admin/health']
       - enabled: only when authenticated

    6. `useSuperAdminErrorLogs()` — GET /api/super-admin/error-logs
       - useQuery: queryKey ['/api/super-admin/error-logs']
       - enabled: only when authenticated

    7. `useSuperAdminSettings()` — GET + PATCH /api/super-admin/company-settings
       - useQuery for GET: queryKey ['/api/super-admin/company-settings'], enabled when authenticated
       - useMutation for PATCH: on success invalidate the settings query key
       - Return both query and mutation
