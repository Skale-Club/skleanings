<objective>
Fix the recurring booking pipeline so that when a customer creates a recurring booking with a non-default duration (e.g. 4 hours instead of the service catalog default of 2 hours), all future generated instances use the same 4-hour duration. This plan makes two surgical fixes: capture the chosen durationMinutes when the subscription is created, and read it in the recurring generator instead of the live catalog value.

Purpose: Without this fix, recurring instances silently revert to the catalog default duration, breaking slot availability, endTime, and totalDurationMinutes for every generated instance after the original. This is the final gap for DUR-06.
Output: Subscription row stores durationMinutes snapshot. Generator reads snapshot with catalog fallback. npm run check and npm run build are green.
</objective>
