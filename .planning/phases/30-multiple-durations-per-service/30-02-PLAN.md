<objective>
Wire the customer's duration selection from the CartContext through the booking API to the database. This plan adds getServiceDuration to the storage interface, fixes getCartItemsForBooking to include selectedDurationId, and updates the booking creation route to resolve the chosen duration and write it as a snapshot into bookingItems rows.

Purpose: After Plan 01 adds the DB columns and Zod field, this plan connects the data plumbing. When a customer selects a 4-hour duration card, that choice now flows end-to-end: CartContext payload → Zod validation → server lookup → bookingItems snapshot columns.
Output: getServiceDuration on IStorage, selectedDurationId in CartContext payload, durationLabel/durationMinutes written to DB on booking creation.
</objective>
