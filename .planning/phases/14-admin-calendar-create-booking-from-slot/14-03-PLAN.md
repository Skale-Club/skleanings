POST /api/bookings — public endpoint (no requireAdmin), accepts JSON body validated by `insertBookingSchema`.

Required request body shape (the literal payload Plan 03 must POST):
```ts
{
  customerName: string,
  customerPhone: string,
  customerAddress: string,         // notNull in schema — must be non-empty
  customerEmail: string | null,    // optional — send empty string as null OR omit
  bookingDate: string,             // "YYYY-MM-DD"
  startTime: string,               // "HH:mm"
  endTime: string,                 // "HH:mm" — computed (override or auto)
  totalDurationMinutes: number,    // service.durationMinutes × quantity (or override-derived)
  totalPrice: string,              // numeric as string — e.g. "120.00"
  paymentMethod: "site",           // D-11
  staffMemberId: number | null,    // optional FK
  cartItems: [{ serviceId: number, quantity: number }],   // D-01: single item
}
```

NOTES:
1. `status` is omitted from `insertBookingSchemaBase` (shared/schema.ts line 455), AND `storage.createBooking` hardcodes `status: 'pending'` (server/storage.ts line 660). To satisfy D-10, after the 201 response do an immediate PATCH `/api/bookings/:id` with `{ status: 'confirmed' }`. The `handleBookingUpdate` route (server/routes/bookings.ts line 204) is `requireAdmin` and accepts partial updates including `status`.
2. `paymentStatus` defaults to `'unpaid'` at the schema level (shared/schema.ts line 177) — D-12 is satisfied automatically. Do NOT send it.
3. `visitorId` is omitted from the request body per D-18.

Response codes:
- 201 → success, body = `Booking` row (use `.id` for the follow-up PATCH)
- 409 → `{ message: "Time slot is no longer available" }` — slot conflict
- 400 → `{ message: 'Validation error', errors: ZodError[] }` (server/routes/bookings.ts line 198) OR `{ message: <error.message> }` for non-Zod errors (line 200)

ZodError shape (from `z.ZodError.errors`):
```ts
Array<{ path: (string | number)[]; message: string; code: string; ... }>
```
The first segment of `path` is the field name to map back to react-hook-form.

Existing helpers in client/src/lib/queryClient.ts:
- `apiRequest(method, url, body?)` — for unauthenticated calls. Returns Response.
- `authenticatedRequest(method, url, token, body?)` — for admin calls. Returns Response.

       const [serverError, setServerError] = useState<string | null>(null);

       const createBookingMutation = useMutation({
         mutationFn: async (payload: any) => {
           const res = await apiRequest('POST', '/api/bookings', payload);
           if (!res.ok) {
             let data: any = null;
             try { data = await res.json(); } catch {}
             const err: any = new Error(data?.message ?? `Request failed (${res.status})`);
             err.status = res.status;
             err.data = data;
             throw err;
           }
           return res.json();
         },
         onSuccess: async (created: { id: number }) => {
           // Honour D-10: status defaults to 'confirmed' for admin-created bookings.
           // storage.createBooking hardcodes 'pending'; PATCH it now. Best-effort: a failure
           // here does not roll back the create — booking exists either way.
           try {
             const token = await getAccessToken();
             if (token) {
               await authenticatedRequest(
                 'PATCH',
                 `/api/bookings/${created.id}`,
                 token,
                 { status: 'confirmed' },
               );
             }
           } catch (patchErr) {
             console.warn('Failed to set booking status to confirmed:', patchErr);
           }

           queryClient.invalidateQueries({ queryKey: ['/api/bookings'] });
           toast({ title: 'Booking created' });
           setNewBookingSlot(null);
           form.reset();
           setServerError(null);
         },
         onError: (err: any) => {
           const status = err?.status as number | undefined;
           const data = err?.data;

           if (status === 400 && Array.isArray(data?.errors)) {
             // Zod field errors → setError on each field (D-17)
             for (const zerr of data.errors) {
               const fieldName = Array.isArray(zerr.path) && zerr.path.length > 0
                 ? String(zerr.path[0])
                 : null;
               if (fieldName && fieldName in form.getValues()) {
                 form.setError(fieldName as any, {
                   type: 'server',
                   message: zerr.message,
                 });
               }
             }
             setServerError(null);
             return;
           }

           // 409 (slot conflict, D-16) or other non-field errors
           setServerError(err?.message ?? 'Failed to create booking');
         },
       });

         if (!selectedService) {

           return;
         }

         // Compute totalDurationMinutes:
         //   - When override is OFF: service.durationMinutes × quantity (already in computedTotalDurationMinutes)
         //   - When override is ON: derive from startTime/endTime difference
         let totalDurationMinutes = computedTotalDurationMinutes;
         if (values.endTimeOverride && values.startTime && values.endTime) {
           const [sh, sm] = values.startTime.split(':').map(Number);
           const [eh, em] = values.endTime.split(':').map(Number);
           const diff = (eh * 60 + em) - (sh * 60 + sm);
           if (diff > 0) totalDurationMinutes = diff;
         }

         const totalPrice = (Number(selectedService.price) * (values.quantity || 1)).toFixed(2);

         const payload = {
           customerName: values.customerName,
           customerPhone: values.customerPhone,
           customerAddress: values.customerAddress,
