# Codebase Structure

**Analysis Date:** 2026-04-04

## Directory Layout

```
skleanings/
├── api/                      # Vercel API handler (generated)
├── client/                    # React frontend
│   └── src/
│       ├── components/        # React components
│       ├── pages/             # Route pages
│       ├── hooks/             # Custom React hooks
│       ├── context/           # React contexts

├── server/                    # Express backend
│   ├── routes/                # API route handlers
│   ├── services/              # Business logic
│   ├── middleware/            # Express middleware
│   └── lib/                   # Server utilities
├── shared/                    # Shared schemas and types
│   ├── models/                # Auth models
│   ├── schema.ts              # Database schemas
│   └── routes.ts              # API route types

├── scripts/                   # Utility scripts
├── migrations/                # Database migrations
├── supabase/                  # Supabase config

```

## Directory Purposes

**client/:**
- Purpose: React frontend application
- Contains: SPA with Vite, routing with wouter, UI with Radix + Tailwind
- Key files: `client/src/App.tsx`, `client/src/main.tsx`, `client/index.html`

**server/:**
- Purpose: Express API server
- Contains: Route handlers, storage layer, services, middleware
- Key files: `server/index.ts`, `server/storage.ts`, `server/routes.ts`

**shared/:**
- Purpose: Shared code between client and server
- Contains: Database schemas (Drizzle), Zod validation schemas, API route types
- Key files: `shared/schema.ts`, `shared/routes.ts`

**script/:**
- Purpose: Build tooling
- Contains: `build.mjs` for client/server bundling
- Key files: `script/build.mjs`

**migrations/:**
- Purpose: Database migration files
- Contains: SQL migration files
- Generated by: Drizzle Kit

## Key File Locations

**Entry Points:**
- `server/index.ts`: Server startup, Express app, route registration
- `client/src/main.tsx`: React app initialization
- `client/index.html`: HTML entry point

**Configuration:**
- `package.json`: Dependencies, scripts, overrides
- `tsconfig.json`: TypeScript config
- `vite.config.ts`: Vite build config
- `tailwind.config.ts`: Tailwind CSS config
- `drizzle.config.ts`: Drizzle Kit config

**Core Logic:**
- `server/storage.ts`: Database CRUD operations (68KB, large file)
- `server/routes.ts`: Route registration logic
- `shared/schema.ts`: Database table definitions (~800 lines)

**Testing:**
- No automated test runner configured per AGENTS.md

## Naming Conventions

**Files:**
- PascalCase for React components: `Admin.tsx`, `BookingPage.tsx`
- camelCase for utilities, hooks: `use-seo.ts`, `analytics.ts`
- kebab-case for config files: `tailwind.config.ts`, `drizzle.config.ts`

**Directories:**
- camelCase: `client/src/components`, `server/routes`
- kebab-case for special: `.github`, `.vercel`

**TypeScript:**
- PascalCase for types: `User`, `Booking`, `CompanySettings`
- camelCase for variables and functions

## Where to Add New Code

**New Feature (Customer-facing):**
- Page component: `client/src/pages/NewFeature.tsx`
- API endpoint: `server/routes/new-feature.ts`
- Schema update: `shared/schema.ts`

**New Admin Panel Section:**
- Page: `client/src/pages/Admin.tsx` (adds new section tab)
- Component: `client/src/components/admin/NewSection.tsx`
- Route handler: `server/routes/admin.ts`

**New Component/Module:**
- UI component: `client/src/components/ui/` or `client/src/components/`
- Shared component: `client/src/components/shared/`
- Server utility: `server/lib/`

**New Database Table:**
- Schema: `shared/schema.ts` (add table definition)
- Storage: `server/storage.ts` (add CRUD functions)
- Route: `server/routes/` (add API endpoints)

**Utilities:**
- Client utilities: `client/src/lib/`
- Server utilities: `server/lib/`
- Shared types: `client/src/types/` or `shared/`

## Special Directories

**dist/:**
- Purpose: Production build output
- Generated: Yes (by `npm run build`)
- Committed: Yes (for Vercel deployment)

**migrations/:**
- Purpose: Database schema migrations
- Generated: Yes (by Drizzle Kit)
- Committed: Yes

**node_modules/:**
- Purpose: Dependencies
- Generated: Yes (by npm install)
- Committed: No (.gitignore)

**.vercel/:**
- Purpose: Vercel deployment config
- Generated: Yes
- Committed: No

**api/:**
- Purpose: Vercel serverless function output
- Generated: Yes (by build)
- Committed: No (in .gitignore typically)
